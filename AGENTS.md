# Project Instructions

- 避免 AI 味明显的表达，不用对立转折句式来包装文案。
- UI 修改默认读取根目录的 `DESIGN.md`。
- 页面风格按 VoltAgent-inspired 方向处理：深色画布、单一电绿强调色、细边框、开发者工具感、代码片段视觉。
- 保持静态站点结构轻量，不引入构建链，除非用户明确要求升级框架。

## Codex Writing Workflow

- 文案任务先读本文件，再读 `DESIGN.md`，最后读取 `docs/voice/` 下的语料文件；没有语料时，明确说明缺少可模仿样本。
- `AGENTS.md` 承载稳定规则，`docs/voice/` 承载历史语料，临时需求放在用户当轮指令里。
- 涉及最近信息、竞品、新闻、产品能力、价格、政策、榜单时，先查证再写，并在回复里说明依据。
- 先确认用户观点或从用户给出的材料中提炼观点，再组织表达；不要替用户编造立场。
- 输出前做一次中文自检，重点检查：物理动作动词滥用、先用形容词下判断、抽象名词当主语、不必要夹英文。
- 保留用户原本的句子节奏、词汇习惯和停顿方式。规则负责减少错误，语料负责贴近本人语气。
- 生成长文时优先输出可直接发布的纯文本；空行、符号、段落长度一次处理好。
