# 视觉系统

## 设计目标

关键词：直白、清晰、标准、商业、年轻、稳重。

整体气质来自两部分：

- 明亮商赛感：白底、亮蓝、橙色、清楚的数据对比。
- 稳重答辩感：深蓝、政策红、规整白卡、明确来源。

## 颜色角色

读取 `../assets/theme-tokens.json` 获取精确色值。

| 颜色 | 用途 |
| --- | --- |
| 深蓝 | 标题、整页案例、整页附录、封面数据栏 |
| 亮蓝 | 市场增长、机会、正向指标、主要数据系列 |
| 浅蓝 | 上期数据、辅助背景、次级信息 |
| 橙色 | 消费、活力、阶段重点、第二数据系列 |
| 浅橙 | 上期数据、辅助比较 |
| 政策红 | 风险、政策、关键警示、高优先级数字 |
| 暖白 | 主要阅读背景 |

同一对象在全篇保持同一颜色含义。

## 背景

### 白底页面

用于市场、用户、商业模式、竞争、行动和大多数分析页。

- 页面背景使用暖白。
- 主要对象使用白卡、细边框和顶部或左侧色条。
- 深蓝只作为文字、细线或底部结论条。
- 禁止在白底中央放大面积孤立深蓝矩形。

### 深蓝整页

用于案例、章节、封面数据栏和资料附录。

- 背景必须覆盖完整区域，形成明确页面模式。
- 内容使用白卡承载，卡片以蓝、橙、红色条区分。
- 正文白色或浅蓝灰，确保对比度。
- 一套 8–12 页演示建议使用 1–3 张深蓝页。

## 字体与层级

中文优先使用 `PingFang SC`、`Microsoft YaHei` 或环境中稳定的无衬线字体。

- 主标题：30–38 px，粗体。
- 封面标题：48–56 px，粗体。
- 副标题：14–18 px，常规。
- 数据：28–48 px，粗体。
- 卡片标题：14–18 px，粗体。
- 正文：12–15 px。
- 页脚：9–11 px。

标题区在各页保持相同左边距和垂直位置。章节标签使用橙色短条配深蓝文字；深色背景改用白字。

## 图表

- 柱状图优先使用 `浅蓝 → 亮蓝` 和 `浅橙 → 亮橙` 对比前后期。
- 风险或显著领先数据可用政策红。
- 直接标注数值和类别，能不用图例就不用。
- 网格线和坐标轴保持浅灰。
- 一张图只证明标题中的一个判断。
- 避免彩虹配色、3D、渐变柱和无意义图标。

## 卡片

- 白卡使用 1 px 浅灰边框。
- 区分类别时使用 6–9 px 左色条或顶色条。
- 同角色卡片保持等高、对齐、内边距一致。
- 圆角控制在 0–8 px，避免软件仪表盘感。
- 卡片数量超过 4 个时，优先改成表格、流程或分组。

## 封面

推荐结构：

- 左侧暖白内容区：标题、副标题、日期、三枚主题标签。
- 右侧规整深蓝数据栏：2–3 个关键数据。
- 两侧可用细红线、橙线或顶部短色带。

禁止使用没有含义的巨大斜切块、随机多边形和重叠渐变。
